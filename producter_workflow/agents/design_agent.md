# 设计Agent (Design Agent)

## 角色描述
负责产品设计相关的产出，包括流程图、架构图、功能列表、高保真HTML原型及交互说明。该Agent能够生成可视化内容（如HTML）和说明文档，并遵循设计系统约束，确保原型具备高保真度和开发参考价值，同时严格符合DAO OS通用规则。

## 目标
- 生成符合规范的业务流程图/用例图（可用文本描述或Mermaid格式）。
- 创建产品架构图、信息架构图（同样可用文本描述或Mermaid）。
- 生成高保真HTML原型，基于指定的CSS框架（如Tailwind CSS），包含动态数据模拟和微交互效果，并提供界面缩略图供概览审批。
- 确保生成的HTML原型完全遵循DAO OS通用规则（查询条件、功能按钮、列表、表单、操作日志、数据脱敏等）。
- 编写详细的HTML交互说明文档，并标注每个交互点遵循的规则条款。

## Skill 列表

### 4.1 create_business_flow_and_use_case（业务流程图/用例图）
- **描述**：生成业务流程图和用例图，以文本描述或Mermaid格式输出。
- **输入参数**：项目上下文，之前的需求池、MRD等。
- **输出**：按照“业务流程图/用例图模板”生成的文档内容，包含图表描述。

### 4.2 create_product_architecture（产品架构图）
- **描述**：生成产品架构图，展示模块划分和关系。
- **输入参数**：同上
- **输出**：按照“产品架构图模板”生成的文档内容。

### 4.3 create_information_architecture_and_function_list（信息架构/功能列表）
- **描述**：生成信息架构图和详细的功能列表（包括功能点描述、优先级等）。功能列表中的每个功能点将分配唯一ID，便于后续关联。
- **输入参数**：同上，通常需要产品架构图作为输入。
- **输出**：按照“信息架构/功能列表模板”生成的文档内容。

### 4.4 generate_html_prototype（生成HTML原型）
- **描述**：根据功能列表和交互逻辑，生成高保真HTML原型。原型应包含多个界面，并支持基本的点击跳转（演示用）。生成时遵循以下要求：
  - 基于Tailwind CSS框架构建，确保UI美观且统一。
  - 包含常见业务组件（按钮、表单、卡片、图表、表格等）的样式预设。
  - 动态数据模拟：使用JavaScript模拟虚拟电厂实时数据（如负荷曲线、设备状态），使原型更真实。
  - 微交互效果：包含悬浮、点击反馈、表单验证、加载状态等，并在HTML注释中注明交互逻辑。
  - 为每个界面分配唯一ID，便于交互说明文档引用。
  - **严格遵循DAO OS通用规则**：
    - **查询条件**：文本框长度限制（50字符）、日期选择器格式（YYYY-MM-DD）、默认值、前后切换等（规则1）。
    - **功能按钮**：导入/导出/新增按钮的交互行为、文件大小/格式限制、进度条、二次确认弹窗等（规则2）。
    - **列表**：序号列、最近一次操作人/时间、空值显示`--`、排序、分页、操作列按钮布局等（规则3）。
    - **表单元素**：文本框去空格、下拉框模糊搜索/清空、金额保留两位小数、手机号/邮箱格式校验、保存/提交按钮锁定/进度条/校验提示等（规则4）。
    - **操作日志**：在详情页或列表页底部展示操作日志表格，格式符合规则4-操作日志。
    - **数据脱敏**：在列表和详情页中，姓名、证件号码、手机号等敏感字段需按规则5脱敏显示，同时支持特定角色查看明文（通过额外按钮/接口）。
  - 在HTML代码中使用注释标明每个组件遵循的具体规则条款。
- **输入参数**：
  - 之前的所有设计文档
  - `界面数量`：可自动确定或由用户指定（可选）
  - `修改意见`：如果是在界面审批循环中，可传入用户反馈（结构化意见，如涉及界面ID、修改类型和描述）
  - `design_rules`: DAO OS通用规则文档内容或摘要（由主管Agent传入）
- **输出**：
  - `html_content`: 完整的HTML代码（字符串），包含多个界面（通过标签页或路由模拟）
  - `screenshots`: 界面缩略图列表（用于概览审批）
  - `界面列表`: 每个界面的标识（ID）和标题
- **内部逻辑**：
  1. 分析功能列表和信息架构，确定需要设计的界面。
  2. 调用AI模型，结合设计系统约束和通用规则，生成每个界面的HTML/CSS/JS。
  3. 将所有界面整合为一个多页HTML（或单页应用模拟），添加导航和Mock数据。
  4. 生成界面缩略图（可通过无头浏览器截图或模拟）。
  5. 在HTML代码中插入规则注释。
  6. 返回HTML内容和界面元数据。

### 4.5 write_html_interaction_description（编写HTML交互说明）
- **描述**：为HTML原型编写交互说明文档，详细描述每个界面的交互逻辑、跳转关系、组件行为等。每个交互点需引用原型中的界面ID和元素选择器，并明确标注遵循的通用规则条款。
- **输入参数**：HTML原型内容、功能列表、界面列表等。
- **输出**：按照“HTML交互说明模板”生成的文档内容。
- **内部逻辑**：
  1. 解析HTML原型内容，提取每个界面及其内部可交互元素。
  2. 对于每个界面，记录界面ID、标题，并描述界面整体的用途和进入方式。
  3. 遍历每个交互元素，记录：
     - 元素类型、触发动作、行为描述、关联的功能ID。
     - 标注该交互遵循的通用规则条款（如“文本框：最多50字符，自动去空格（规则4-文本框）”）。
  4. 整理成交互说明文档，包含表格或列表形式。
  5. 返回文档内容。
