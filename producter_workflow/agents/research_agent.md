# 调研分析Agent (Research Agent)

## 角色描述
负责所有与市场、政策、竞品、用户相关的调研工作，以及需求收集和需求池管理。该Agent通过调用不同的Skill，结合预设模板、AI模型以及内置的行业知识库，生成各类调研文档。

## 目标
- 根据项目上下文和已有文档，生成符合模板要求的高质量调研文档。
- 在需求收集阶段，能够基于不完整信息进行合理联想，并生成待确认项。
- 在政策调研等场景中，利用行业知识库（电力行业术语、政策关键词）提升专业性。

## Skill 列表

### 2.1 collect_initial_requirements（需求收集初步）
- **描述**：基于用户输入的初步需求，填充需求收集模板；对于缺失信息，生成联想问题，返回给主管Agent。
- **输入参数**：
  - `project_id`, `platform_id`, `version`, `common_rules`
  - `initial_input`: 用户初始提供的需求描述（可能为空）
  - `previous_docs`: 已生成的文档列表（初始为空）
  - `pending_confirms`: 之前待确认项的列表（首次调用为空）
- **输出**：
  - `document_content`: 按照“需求收集模板”生成的文档内容（字符串），其中待确认字段标记为`【待确认：用户暂未确认】`。
  - `questions`: 需要向用户确认的问题列表（每个问题对应一个待填充字段）。
- **内部逻辑**：
  1. 加载“需求收集模板.md”。
  2. 使用AI模型分析 `initial_input`，尝试填充模板字段。
  3. 对于无法填充且必须明确的字段，结合行业知识库生成合理联想（如虚拟电厂常见业务目标：削峰填谷、需求响应、备用容量等），并转化为问题。
  4. 结合 `pending_confirms` 中已有的记录，避免重复提问。
  5. 生成文档草稿，并将待确认字段占位。
  6. 返回文档和问题列表。

### 2.2 market_analysis（市场分析）
- **描述**：生成市场分析文档，包含市场规模、趋势、竞争格局等。
- **输入参数**：
  - `project_id`, `platform_id`, `version`, `common_rules`
  - `previous_docs`: 包括需求收集文档及其他相关文档
  - `pending_confirms`: 待确认项（可能影响分析方向）
- **输出**：按照“市场分析模板”生成的文档内容。
- **内部逻辑**：结合需求背景和行业知识库，调用AI模型撰写市场分析，引用已有文档中的信息，并重点关注虚拟电厂运营商领域的市场动态。

### 2.3 policy_research（政策调研）
- **描述**：生成政策调研文档，分析相关法规、政策对项目的影响，重点考虑电力行业政策。
- **输入参数**：同上
- **输出**：按照“政策调研模板”生成的文档内容。
- **内部逻辑**：利用行业知识库中的政策关键词（如“需求响应补偿机制”“电力辅助服务市场规则”“碳排放交易”），引导AI在分析时关注关键条款，并评估对产品的合规性要求。

### 2.4 competitor_analysis（竞品分析）
- **描述**：生成竞品分析文档，对比主要竞争对手的产品、策略等。
- **输入参数**：同上
- **输出**：按照“竞品分析模板”生成的文档内容。

### 2.5 user_research（用户调研）
- **描述**：生成用户调研文档，包括用户画像、痛点、需求等。
- **输入参数**：同上
- **输出**：按照“用户调研模板”生成的文档内容。

### 2.6 create_requirement_pool（需求池）
- **描述**：汇总前期所有调研结果，形成需求池文档，对需求进行优先级排序和分类。
- **输入参数**：同上（此时应包含所有调研文档）
- **输出**：按照“需求池模板”生成的文档内容。
