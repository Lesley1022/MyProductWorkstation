# Skill: collect_initial_requirements

## 所属Agent
调研分析Agent (Research Agent)

## 描述
基于用户初步输入的需求描述，填充“需求收集模板”。对于模板中无法根据输入填充的字段，该技能会结合行业知识库生成合理的联想，并转化为待确认问题列表返回。该技能支持待确认项的累积，避免重复提问。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识，如 "dao_os_trade" |
| `platform_id` | string | 是 | 平台标识，如 "taiqi_xiaoyuanzhidian" |
| `version` | string | 是 | 版本号，如 "V2.1.0.c" |
| `common_rules` | object | 否 | DAO OS通用规则文档内容或引用，用于后续步骤的规则遵循 |
| `initial_input` | string | 否 | 用户初步提供的需求描述文本，可能为空 |
| `previous_docs` | array | 是 | 已生成并通过审核的文档列表（每个元素包含文档类型和内容），初始为空 |
| `pending_confirms` | array | 是 | 之前步骤中用户未确认的待确认项列表，每个元素为 `{ field: string, question: string }` |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“需求收集模板”生成的完整文档内容，其中待确认字段标记为 `【待确认：用户暂未确认】` |
| `questions` | array | 需要向用户确认的问题列表，每个元素为 `{ field: string, question: string, options?: string[] }`，其中options为AI联想的可能选项 |

## 内部逻辑
1. 加载“需求收集模板”文件。
2. 使用AI模型分析 `initial_input`，尝试填充模板中的字段。
3. 遍历模板中所有字段，识别未能填充的字段。
4. 对于每个未填充字段，结合内置的虚拟电厂行业知识库（包含常见业务目标、用户类型、资源类型等）生成合理的联想选项。
5. 检查 `pending_confirms` 中是否已有该字段的记录，若有则跳过（避免重复提问）。
6. 生成每个未填充字段的确认问题，并提供联想选项（若有）。
7. 将已填充的内容和待确认标记合并到模板中，生成完整的文档内容。
8. 返回文档内容和问题列表。

## 注意事项
- 生成的文档中待确认字段需要特殊标记，便于后续步骤识别。
- 问题列表中的每个问题应清晰表述，并提供选项供用户选择，若用户拒绝提供则标记为待确认。
