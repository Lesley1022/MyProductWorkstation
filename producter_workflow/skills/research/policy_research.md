# Skill: policy_research

## 所属Agent
调研分析Agent (Research Agent)

## 描述
生成政策调研文档，分析相关国家/地区/行业的政策法规对项目的影响。重点考虑虚拟电厂领域的电力市场政策、需求响应机制、碳排放交易等。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，至少包含需求收集文档 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“政策调研模板”生成的完整文档内容 |

## 内部逻辑
1. 加载“政策调研模板”。
2. 从 `previous_docs` 中提取项目涉及的地域、业务类型等信息。
3. 调用内置的政策知识库（可配置的电力行业政策关键词库），检索相关的最新政策动态。
4. 使用AI模型分析政策对产品的影响（机会与约束），包括：
   - 合规性要求（如数据隐私、调度规则）
   - 激励政策（如补贴、需求响应补偿）
   - 准入门槛
5. 生成文档内容，并引用相关政策文件或条款。
6. 若项目地域未明确，则根据待确认项提示用户补充。

## 注意事项
- 政策信息应尽量引用官方来源，如无法获取，应注明“需进一步调研”。
- 需区分国家政策和地方政策，确保分析具有针对性。
