# Skill: create_requirement_pool

## 所属Agent
调研分析Agent (Research Agent)

## 描述
汇总前期所有调研结果（用户调研、竞品分析、政策分析等），形成需求池文档。需求池包含需求ID、需求描述、来源、优先级、状态等信息。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，应包含所有调研文档及需求收集文档 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“需求池模板”生成的完整文档内容 |

## 内部逻辑
1. 加载“需求池模板”。
2. 从所有调研文档中提取需求描述，包括：
   - 用户调研中的用户需求
   - 竞品分析中的功能差距
   - 政策调研中的合规要求
   - 市场分析中的机会点
3. 对每个需求进行去重、合并，生成唯一需求ID。
4. 为每个需求标注来源（如“用户调研-运营商”）。
5. 使用MoSCoW方法（必须有、应该有、可以有、不会有）对需求进行优先级排序，并结合业务目标。
6. 将需求归类（如功能需求、非功能需求、数据需求）。
7. 生成需求池表格，包含字段：ID、需求名称、描述、来源、优先级、类别、状态（初始为“待评审”）。
8. 返回文档内容。

## 注意事项
- 需求池是后续设计的输入，需确保覆盖全面且优先级明确。
- 若存在待确认项，应在需求描述中注明“待确认”。
