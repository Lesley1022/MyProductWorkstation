# Skill: user_research

## 所属Agent
调研分析Agent (Research Agent)

## 描述
生成用户调研文档，包含用户画像、用户旅程、痛点分析、需求提炼。帮助产品团队理解目标用户。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，至少包含需求收集、市场分析 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“用户调研模板”生成的完整文档内容 |

## 内部逻辑
1. 加载“用户调研模板”。
2. 从需求收集文档中提取目标用户描述（如虚拟电厂运营商、负荷聚合商、终端用户等），若未明确则根据待确认项生成假设。
3. 为每种用户角色创建用户画像，包括：
   - 基本信息（角色、职责）
   - 目标与动机
   - 痛点与挑战
   - 使用场景
4. 绘制用户旅程图（文字描述或Mermaid），展示用户与产品交互的关键阶段。
5. 提炼用户需求，并与功能列表初步对应。
6. 返回生成的文档内容。

## 注意事项
- 若缺乏一手调研数据，应注明“基于行业经验假设”，并建议后续验证。
- 用户旅程应真实反映业务场景。
