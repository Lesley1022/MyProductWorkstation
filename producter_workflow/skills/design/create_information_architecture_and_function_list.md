# Skill: create_information_architecture_and_function_list

## 所属Agent
设计Agent (Design Agent)

## 描述
生成信息架构图和详细的功能列表。信息架构图展示产品信息层级和导航结构，功能列表包含每个功能的ID、名称、描述、优先级，并关联需求池中的需求ID。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，至少包含产品架构图、需求池 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“信息架构/功能列表模板”生成的文档内容，包含信息架构图（Mermaid）和功能列表表格 |

## 内部逻辑
1. 加载“业务流程图与用例图模板”文件。
2. 从产品架构图中提取模块，设计信息层级，如：
   - 一级导航：仪表盘、资源管理、交易市场、报表中心、系统设置
   - 二级页面：资源列表、资源详情、新建资源等
3. 使用Mermaid的`graph LR`绘制信息架构图。
4. 根据需求池，细化每个页面/模块的功能点，生成功能列表，每行包含：
   - 功能ID（格式：FUNC-XXX）
   - 功能名称
   - 功能描述
   - 优先级（P0/P1/P2）
   - 关联需求ID
5. 返回文档内容。

## 注意事项
- 功能ID需唯一，后续所有文档（PRD、交互说明、埋点）都将引用此ID。
- 功能列表应覆盖所有需求池中的功能需求。
