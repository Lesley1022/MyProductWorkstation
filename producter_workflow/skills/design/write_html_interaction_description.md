# Skill: write_html_interaction_description

## 所属Agent
设计Agent (Design Agent)

## 描述
为HTML原型编写交互说明文档，详细描述每个界面的交互逻辑、跳转关系、组件行为、输入输出等。文档将关联功能ID和界面ID，供PRD引用，并明确标注每个交互点遵循的通用规则条款。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 是 | DAO OS通用规则文档内容 |
| `previous_docs` | array | 是 | 已生成的文档列表，必须包含HTML原型内容（作为字符串）、功能列表、界面列表 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“HTML交互说明模板”生成的完整文档内容 |

## 内部逻辑
1. 加载“HTML交互说明模板”。
2. 解析HTML原型内容，提取每个界面及其内部可交互元素（按钮、表单、链接等）。
3. 对于每个界面：
   - 记录界面ID和标题。
   - 描述界面整体的用途和进入方式。
   - 遍历界面内的每个交互元素，记录：
     - 元素类型（按钮、输入框等）
     - 触发动作（点击、输入、悬停）
     - 行为描述（如“点击‘提交’按钮后，验证表单，若成功则跳转到详情页”）
     - 关联的功能ID（从HTML注释或功能列表推断）
     - 标注该交互遵循的通用规则条款（如“文本框：最多50字符，自动去空格（规则4-文本框）”）
4. 整理成交互说明文档，包含表格或列表形式。
5. 返回文档内容。

## 注意事项
- 交互说明应覆盖所有原型中的交互点，不留死角。
- 关联功能ID至关重要，确保PRD能够引用。
- 规则标注需准确，便于开发人员理解规范要求。
