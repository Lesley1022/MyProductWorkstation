# Skill: create_product_architecture

## 所属Agent
设计Agent (Design Agent)

## 描述
生成产品架构图，展示产品模块划分、技术栈、数据流向等。用于指导后续详细设计和开发。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，至少包含业务流程图/用例图、需求池 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“产品架构图模板”生成的文档内容，包含架构图（Mermaid/文本）和说明 |

## 内部逻辑
1. 加载“产品架构图模板”。
2. 分析需求池和业务流程图，识别主要功能模块（如资源管理、交易中心、数据分析、用户中心）。
3. 确定模块间依赖和数据交互关系。
4. 使用Mermaid的`graph TD`或`flowchart`绘制架构图，包括前端、后端、数据库、外部系统（如电网调度系统）。
5. 描述每个模块的职责和技术选型建议。
6. 返回文档内容。

## 注意事项
- 架构图应保持高层级，不涉及内部组件细节。
- 技术选型建议基于常见实践，但最终由开发团队决定。
