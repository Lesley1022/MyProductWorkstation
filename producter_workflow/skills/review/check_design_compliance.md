# Skill: check_design_compliance

## 所属Agent
审核Agent (Review Agent)

## 描述
专门用于检查HTML原型是否符合DAO OS通用规则文档，返回不符合项列表。可作为`review_document`的辅助或独立调用。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `html_content` | string | 是 | HTML原型内容 |
| `rules_doc` | object | 是 | 规则文档（或结构化规则集） |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `passed` | boolean | 是否全部符合 |
| `violations` | array | 不符合项列表，每个元素包含 `element`（元素描述）、`rule`（违反的规则条款）、`suggestion`（修改建议） |

## 内部逻辑
1. 解析HTML内容，提取所有可交互元素（如输入框、按钮、表格等）。
2. 遍历规则集，对每个元素进行规则匹配：
   - 检查文本框是否有 `maxlength` 属性且≤50。
   - 检查日期输入格式是否为 `YYYY-MM-DD`。
   - 检查列表是否包含序号列、最近一次操作人/时间列。
   - 检查敏感字段的文本是否已脱敏（如手机号显示为139****8978）。
   - 检查删除/停用按钮是否有 `confirm` 弹窗相关JS。
   - 检查保存/提交按钮是否包含进度条和锁定逻辑。
   - 检查操作日志表格是否存在。
3. 汇总不符合项，生成详细描述和修改建议。
4. 返回结果。

## 注意事项
- 该技能需要规则集结构化，以便程序化检查。
- 对于复杂规则（如业务逻辑），可能仍需人工判断，但基础UI规则可自动化。
