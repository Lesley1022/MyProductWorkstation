# Skill: review_document

## 所属Agent
审核Agent (Review Agent)

## 描述
对指定类型的文档进行内部评审，检查其完整性、格式规范性、与上下文的逻辑一致性，以及是否符合DAO OS通用规则（如适用）。返回是否通过及修改建议。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `document_type` | string | 是 | 文档类型标识，如 "需求收集"、"市场分析"、"BRD"、"HTML原型" 等 |
| `document_content` | string | 是 | 待评审的文档内容 |
| `context` | object | 是 | 项目上下文，包括 `previous_docs`（已通过审核的文档列表）、`pending_confirms`、`common_rules`（通用规则文档） |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `passed` | boolean | 是否通过内部评审 |
| `comments` | array | 评审意见列表，每个元素为 `{ severity: "error" | "warning" | "info", message: string }` |

## 内部逻辑
1. 根据 `document_type` 加载对应的评审规则集（可配置）。规则包括：
   - 模板字段完整性检查：是否所有必填章节都已填写。
   - 格式规范：是否符合Markdown标题层级、表格格式等。
   - 一致性检查：与已有文档是否存在矛盾（例如PRD中的功能与功能列表不一致）。
   - **通用规则符合性检查**（针对HTML原型和PRD）：
     - 对于HTML原型，检查：
       - 所有文本框是否设置了`maxlength=50`。
       - 日期选择器格式是否为`YYYY-MM-DD`。
       - 列表是否有序号列、最近一次操作人/时间列。
       - 敏感字段是否按规则5脱敏。
       - 删除/停用等操作是否有二次确认弹窗。
       - 保存/提交按钮是否有进度条和锁定效果。
       - 操作日志是否按规则4展示。
     - 对于PRD文档，检查：
       - 是否在功能需求中引用了通用规则。
       - 非功能性需求中是否包含了数据脱敏、操作日志等要求。
       - 技术规格中是否有“数据脱敏实现”章节。
2. 使用AI模型辅助判断内容逻辑是否通顺、合理。
3. 汇总违反规则的项，生成评审意见。
4. 若存在严重错误（如缺失关键章节），则 `passed` 为 false；若仅轻微建议，则 `passed` 为 true 但仍返回意见。
5. 返回评审结果。

## 注意事项
- 评审规则应随项目经验持续优化。
- 对于待确认项，评审时应适当放宽要求，但需在意见中提醒。
