# Skill: tech_review

## 所属Agent
审核Agent (Review Agent)

## 描述
对PRD文档进行技术预审，模拟开发团队视角，识别实现难点、技术风险、遗漏的技术考量（如性能、安全、扩展性）。返回技术评审意见。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `prd_content` | string | 是 | PRD文档内容 |
| `context` | object | 是 | 项目上下文，包括产品架构图、功能列表等 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `passed` | boolean | 是否通过技术预审（无重大技术风险） |
| `tech_comments` | array | 技术层面的建议或风险提示列表，每个元素为 `{ severity: "high" | "medium" | "low", message: string }` |

## 内部逻辑
1. 解析PRD中的功能需求和非功能需求。
2. 结合产品架构图，评估每个功能的技术可行性：
   - 是否存在过于复杂或不切实际的交互？
   - 性能指标是否合理（如并发量、响应时间）？
   - 数据模型是否完整，有无遗漏字段？
   - 是否依赖外部系统，接口定义是否清晰？
   - 安全要求是否考虑（如数据加密、权限控制）？
3. 识别潜在技术风险，如实时数据处理、大数据量存储、第三方集成可靠性。
4. 给出技术预审意见，并提出改进建议。
5. 返回结果。

## 注意事项
- 该技能为可选，可在PRD审核前调用，帮助提前发现技术问题。
- 评审意见应具体可操作，避免空泛。
