# Skill: write_prd

## 所属Agent
文档撰写Agent (Document Writer Agent)

## 描述
生成产品需求文档（PRD），详细描述产品功能、交互逻辑、非功能需求、技术规格等。PRD是开发团队的主要参考文档，需与原型、交互说明保持一致，并明确引用DAO OS通用规则。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 是 | DAO OS通用规则文档内容，必须传入以确保引用 |
| `previous_docs` | array | 是 | 已生成的文档列表，必须包含MRD、业务流程图/用例图、产品架构图、信息架构/功能列表、HTML交互说明、HTML原型 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“PRD模板”生成的完整文档内容，包含对通用规则的引用 |

## 内部逻辑
1. 加载“PRD模板”。
2. 从 `previous_docs` 中提取信息：
   - 功能需求：从功能列表中提取每个功能的详细描述，结合HTML交互说明中的交互细节。对每个功能点，从HTML原型中截取对应的界面图片嵌入文档。
   - 在功能需求中，明确标注每个界面或组件遵循的DAO OS通用规则条款（如“列表排序规则见规则3”）。
   - 非功能需求：包括性能指标（响应时间、并发用户）、安全性、可用性等，并增加对通用规则的遵循要求：
     - 数据脱敏：按规则5在存储层实现脱敏。
     - 操作日志：按规则4记录所有操作。
     - 权限控制：按规则3-数据权限实现。
   - 技术规格：根据产品架构图和功能列表，推导出接口定义（RESTful API）、数据模型（字段、类型）、权限矩阵、异常处理等。新增“数据脱敏实现”小节，说明存储层脱敏方案（如哈希、加密、掩码），并列出各字段的脱敏规则（规则5）。明确明文查看的权限控制和日志记录要求。
3. 确保每个功能点都有唯一ID，并与原型中的元素对应。
4. 生成文档内容，并返回。

## 注意事项
- PRD必须与原型严格一致，任何不一致都可能导致开发误解。
- 技术规格章节应尽可能详细，减少开发过程中的沟通成本。
- 必须引用通用规则，确保功能实现符合平台规范。
