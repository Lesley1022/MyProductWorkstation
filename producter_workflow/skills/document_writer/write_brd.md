# Skill: write_brd

## 所属Agent
文档撰写Agent (Document Writer Agent)

## 描述
生成商业需求文档（BRD），阐述项目背景、商业目标、预期收益、风险评估等。用于向决策层说明项目的商业价值。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，至少包含需求收集、市场分析、政策调研、竞品分析、用户调研 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“BRD模板”生成的完整文档内容 |

## 内部逻辑
1. 加载“BRD模板”。
2. 从 `previous_docs` 中提取关键信息：
   - 项目背景：从需求收集、市场分析、政策调研中提取市场机会、政策驱动等。
   - 商业目标：结合需求收集中的业务目标，设定可量化的目标（如市场份额、营收增长）。
   - 预期收益：分析ROI、成本节约、效率提升等。
   - 风险评估：识别政策风险、技术风险、市场风险，并提出缓解措施。
3. 使用AI模型组织语言，确保逻辑清晰、数据支撑充分。
4. 生成完整的BRD文档。
5. 返回文档内容。

## 注意事项
- BRD应聚焦商业层面，避免过多功能细节。
- 若存在待确认项，需在文档中说明假设条件。
