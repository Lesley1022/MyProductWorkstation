# Skill: write_mrd

## 所属Agent
文档撰写Agent (Document Writer Agent)

## 描述
生成市场需求文档（MRD），描述目标市场、用户需求、产品定位、功能概要、市场策略等。用于指导产品方向和市场进入。

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `project_id` | string | 是 | 项目标识 |
| `platform_id` | string | 是 | 平台标识 |
| `version` | string | 是 | 版本号 |
| `common_rules` | object | 否 | DAO OS通用规则文档 |
| `previous_docs` | array | 是 | 已生成的文档列表，至少包含BRD及所有调研文档 |
| `pending_confirms` | array | 是 | 待确认项列表 |

## 输出参数
| 参数名 | 类型 | 描述 |
|--------|------|------|
| `document_content` | string | 按照“MRD模板”生成的完整文档内容 |

## 内部逻辑
1. 加载“MRD模板”。
2. 从 `previous_docs` 中提取：
   - 目标市场：从市场分析中提取细分市场、市场规模。
   - 用户需求：从用户调研、需求池中提取核心用户需求。
   - 产品定位：基于竞品分析和产品优势，确定差异化定位。
   - 功能概要：从需求池中提取高优先级功能，形成功能模块列表。
   - 市场策略：包括定价、推广渠道、合作伙伴等初步设想。
3. 使用AI模型整合内容，形成连贯的MRD。
4. 返回文档内容。

## 注意事项
- MRD是连接商业和产品的桥梁，需兼顾商业可行性和用户价值。
- 功能概要应保持高层级，不涉及详细交互。
