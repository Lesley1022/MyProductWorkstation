# DAO OS 通用设计规范（可引用版）

适用范围：DAO OS 全系统。
规则原则：若模块无特殊规则，必须遵循本规范；若有特殊规则，必须在模块设计文档或 PRD 中明确“覆盖条款ID + 覆盖原因”。

## 0. 引用说明

为便于在 HTML 原型、PRD、评审意见中直接引用，本规范为每条规则分配唯一 ID。

引用格式建议：
- `按规则 [Q-DATE-001] 处理日期区间：不允许跨月。`
- `本页导入交互遵循 [BTN-IMPORT-004]，导入中锁屏并显示进度。`
- `脱敏规则按 [MASK-ADDR-001] 执行。`

## 5. 数据脱敏规则

原则：明文查看仅限有权限角色；脱敏不应只做界面层，存储层必须配合哈希/掩码/加密。

| 规则ID | 字段名称 | 脱敏规则 |
| --- | --- | --- |
| MASK-NAME-001 | 姓名 | 两位姓名脱敏第二位（张*）；三位及以上显示首尾（张**三）。 |
| MASK-ID-001 | 证件号码 | 显示第一位和最后一位，如 `3**************4`。 |
| MASK-BANK-001 | 银行卡号 | 仅显示后四位，如 `*********2345`。 |
| MASK-PHONE-001 | 手机号 | 显示前三后四位，如 `139****8978`。 |
| MASK-EMAIL-001 | 电子邮箱 | 显示第一位和 @ 后内容，如 `z***@example.com`。 |
| MASK-ADDR-001 | 地址 | 保留省/市/区（县），街道与门牌等详细地址脱敏，如 `安徽省合肥市蜀山区*******`。 |

## 6. HTML/PRD 引用清单（建议）

在 HTML 原型或 PRD 里，至少显式引用以下规则组：
- 查询条件：`Q-*`
- 按钮行为：`BTN-*`
- 列表行为：`LIST-*`
- 表单与提交：`UI-*`
- 脱敏与日志：`MASK-*`、`LOG-*`

推荐在文档附录提供“规则引用矩阵”：
- 列1：页面/模块
- 列2：规则ID
- 列3：是否满足
- 列4：特殊覆盖说明
