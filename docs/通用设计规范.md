# 通用设计规范（精简版）

适用范围：除模块特殊说明外，默认遵循本规范。

## 1. 查询与筛选
- 文本输入：最长 50 字符，支持模糊匹配。
- 日期：格式 `YYYY-MM-DD`，默认不超过当前日期。
- 默认操作：必须提供“查询”和“重置”。
- 空结果提示：`暂无符合查询条件的数据`。

## 2. 列表与分页
- 必备列：序号、核心业务字段、操作列。
- 空值统一显示：`--`。
- 支持分页：默认 10 条/页，可切换 10/15/20/50/100。
- 成功操作（新增/编辑/删除/启停）后必须给出 toast 提示。

## 3. 表单与校验
- 必填项：未填写时字段标红并提示“请填写”。
- 格式错误：字段标红并提示“请正确填写”。
- 保存/提交时需显示处理中状态，避免重复点击。
- 异常提示需包含“原因+下一步建议”。

## 4. 文件上传
- 默认支持：
  - 图片：`jpg/png`，单文件 <= 1MB。
  - 附件：`doc/docx/xls/xlsx/pdf/txt/zip/rar`，单文件 <= 10MB。
- 实际限制可在具体模块中覆盖。

## 5. 数据与权限
- 查询和操作均受角色权限控制。
- 敏感信息默认脱敏展示，按角色开放明文查看。
- 推荐脱敏：手机号前三后四、邮箱首字母+域名、银行卡仅后四位。

## 6. 非功能基线
- 性能：核心列表查询响应建议 <= 3s。
- 安全：关键操作留审计日志（操作者、时间、动作、结果）。
- 可维护：关键流程需可追踪（日志、错误码、告警）。
