# 能源市场分析 Skill

当你需要撰写电力市场分析报告时，必须遵循以下规范。

## ⚙️ 强制规则
- **必须**：包含政策/规则列表、影响分析、机会与风险、应对建议。
- **必须**：基于权威来源（政府官网、电力交易中心、行业协会）。
- **必须**：文件名格式：`能源市场分析_{{YYYYMMDD}}.md`。

## 报告结构
1. **分析范围**：本次分析覆盖的政策领域、交易品种或市场区域。
2. **政策/规则列表**：以表格形式列出相关政策名称、发布机构、核心条款。
3. **影响分析**：分析每项政策或规则对产品功能、技术实现、商业模式的影响。
4. **机会与风险识别**：总结政策带来的市场机会和潜在风险。
5. **应对建议**：产品应如何调整功能、策略以抓住机会、规避风险。

## 🤖 执行指令
- 必须阅读原始文档，不得凭空捏造。
- 可附上原文链接或截图。
## 政策调研生成注意事项（不写入模板）
- 政策原文截图 + 标注：文档中应附关键条款原文截图，并用高亮/框选标注核心句，让读者一眼定位重点。
- 条款分类表格：必须用表格对关键条款按“强制性要求/鼓励性方向/量化指标/职责分工”分类梳理，避免散点叙述。
- 时间轴可视化：对发布日期、生效日、截止日、验收节点使用时间轴展示，突出时限压力与执行节奏。
- 影响链路流程图：使用流程图展示“政策条款 -> 业务环节 -> 产品模块 -> 角色职责 -> 验收点”的波及链路。
- 术语解释清晰：首次出现的政策术语必须给出简明释义，避免跨部门认知偏差。
- 协作留痕：记录跨部门协作过程（参与部门、结论、责任人、时间），并明确责任边界。
- 结论先行（摘要写什么）：长文首页需提供 Executive Summary，至少包含：
  - 政策核心意图与监管方向（一句话）
  - 不可触碰红线（强制条款）
  - 关键时间节点与最后期限
  - 对产品/市场的首要影响
  - 建议动作与优先级（P0/P1/P2）
- 本节仅约束“生成政策调研文档”的表达方式，不得直接写入模板正文。
