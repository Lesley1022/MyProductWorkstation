# 设计评审 Skill

## 评审维度
- [ ] **需求完整性**：是否覆盖所有用户角色、使用场景、异常流程？
- [ ] **逻辑一致性**：功能描述是否前后一致？是否存在矛盾？
- [ ] **用户体验**：交互流程是否直观？是否有不必要的步骤？
- [ ] **界面一致性**：是否符合设计规范（配色、字体、间距）？
- [ ] **可访问性**：颜色对比度、字体大小是否合适？
- [ ] **错误处理**：是否有适当的错误提示和预防机制？
- [ ] **边界情况**：极端数据或操作是否考虑？
- [ ] **可测试性**：验收条件是否明确、可量化？
- [ ] **行业逻辑一致性**：结算公式、工单状态流转是否符合业务规则？
- [ ] **数据可视化规范性**：曲线图、仪表盘是否按行业标准展示（如坐标轴标签、单位、阈值着色）？
- [ ] **政策合规性**：设计是否满足最新政策要求（如并网数据上报）？

## 评审报告格式

- ...

3. **[P2] 可后续考虑**：...

**总结**：
- ...
