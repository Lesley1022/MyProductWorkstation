# 市场需求文档 (MRD) Skill

## 强制规则
- **必须**：严格按下方结构输出，不得增减章节。
- **必须**：所有 `{{占位符}}` 替换为实际内容，未知用“待补充”。
- **必须**：文件名格式：`MRD_{{YYYYMMDD}}.md`（建议可选包含平台标识）。
- **必须**：在“目标用户”中细化电力行业角色：如资产方运营人员、运维方调度员、财务结算专员。
- **必须**：在“用户需求”中要求结合研究员报告中的用户反馈，提炼典型场景：如光伏板清洗工单派发、储能SOC异常告警处理、分时结算报表导出。
- **必须**：在“功能概览”中强制列出两个平台的核心模块，并注明归属。

## 1. 目标市场
- 目标用户群体细分（含电力行业角色）。

## 2. 用户需求
- 通过研究提炼的核心需求（结合研究员报告，提炼典型场景）。

## 3. 产品定位
- 产品在市场中的定位。

## 4. 功能概览
- 主要功能模块列表，并注明各模块归属平台（asset-management/oam-collaboration）。

## 5. 发布策略
- 分阶段发布计划。

## 执行指令
- 必须遵循上述结构和规则。
## 市场需求分析（MRD）生成注意事项（不写入模板，V2 以本节为准）
- 基础信息完整：文档首页必须完整包含名称、版本、拟稿人、审核人。
- 承接 BRD：项目背景与目标必须与 BRD 结论衔接，明确产品定位与核心价值主张。
- 多角色画像卡片：目标用户必须用画像卡片呈现，至少覆盖决策者、使用者、影响者。
- 场景故事化：用户场景需用故事形式描述，并引用用户原声增强共情。
- 旅程情绪曲线：用户旅程建议配情绪曲线，明确体验低谷与触发点。
- 需求功能映射表：必须使用表格展示“用户需求 -> 对应功能 -> 优先级（P0/P1/P2）”。
- 功能全景可视化：功能全景图需用思维导图或结构化列表展示模块关系。
- 核心指标双维度：指标必须同时覆盖用户侧（留存/NPS）与业务侧（付费转化/客单价）。
- 上线标准分层：成功标准需明确“必须达成（底线）”与“期望达成（挑战）”。
- 术语清晰：首次出现术语需简明解释，避免跨团队认知偏差。
- 结论先行（摘要写什么）：长文首页 Executive Summary 至少包含核心用户、核心场景、核心功能、成功标准。
- 覆盖规则：如与旧版 MRD 生成注意事项冲突，以本节（V2）为准。
- 本节仅约束“生成市场需求分析（MRD）文档”的表达方式，不得直接写入模板正文。
